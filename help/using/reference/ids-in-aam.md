# Index des identifiants dans [!DNL Audience Manager] {#index-of-ids-in-audience-manager}

## Présentation {#overview}

[!DNL Audience Manager] utilise plusieurs identifiants pour identifier et gérer les données que vous lui envoyez. Reportez-vous à cet article pour obtenir la liste complète des identifiants [!DNL Audience Manager], ainsi que des exemples des préfixes avec lesquels vous devez les utiliser.

## Préfixe d’ID {#prefixing}

Bien que vous puissiez faire référence à la plupart de ces identifiants par leurs noms autonomes, la plupart d’entre eux sont destinés à être utilisés avec divers préfixes lors de la transmission de données par le biais d’appels [!DNL DCS]. Certains de ces identifiants sont utilisés par [!DNL Audience Manager] sans être exposés aux utilisateurs, tandis que d’autres sont également visibles dans l’interface utilisateur.

Pour comprendre les préfixes utilisés dans les exemples suivants, voir [Attributs pris en charge pour les appels API DCS](../api/dcs-intro/dcs-api-reference/dcs-keys.md).

## [!DNL Global Device IDs] {#global-device-ids}

Le tableau ci-dessous explique ce que sont ces identifiants et leur format. Pour plus d’informations sur les identifiants globaux d’appareil et sur leur utilisation dans [!DNL Audience Manager], consultez la section [Sources de données globales](/help/using/features/global-data-sources.md).
